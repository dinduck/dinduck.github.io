第一个电路

  • 新建一个文件 MAC.hs
  • 第一行代码
module MAC where

定义一个名为 MAC 的模块

  • 引入 Clash.Prelude
import Clash.Prelude

引入 Clash 通用的一些模块, 例如 Sinal, NFDataX 等等

最后我们开始第一句代码

ma acc (x, y) = acc + x * y

我们定义一个名为 ma 的函数， 显然目前还用上 Clash 的功能， 稍后我们会处理。

进入 clashi 交互界面加载 MAC.hs 文件

目前为止， 至少没报错， 但是这里并没有使用上 Clash 的任何东西。

同步时序电路

我们编写一个符合 mealy 机器的函数, 我们可以从官方参考文档找到相关信息。

他的用途是 state -> input -> (new_state, output)

输入当前状态和输入返回新的状态和输出

编辑 MAC.hs

macT acc (x, y) = (acc', o)
  where
    acc' = ma acc (x, y)
    o    = acc

显然我们也可以这样去写 MAC.hs

macT acc inp = (ma acc inp, acc)

Haskell 会自动推断 inp 为 Num a => (a, a)

观察一下 macT 的类型

现在开始使用 Clash 的东西， 也就是 mealy， 其实就是一句话

mac xy = mealy macT 0 xy

我们又新建一个函数， 他绑定了 mealy macT 0 xy, 在这里 ma 和 macT 函数都没有对参数的类型进行约束， 但是 mealy 在这里就会对 macT 的参数进行约束。

对我们来说，只需要关心流程， 最后我们使用 Clash 约束我们的逻辑， 使其为数字电路的逻辑范畴。

测试以下我们的函数功能

>>> import qualified Data.List as L
>>> L.take 4 $ simulate @System mac [(1,1),(2,2),(3,3),(4,4)]
[0, 1, 5, 14]

生成 VHDL

生成 VHDL 的命令是 :vhdl

这里说没有 topEntity 函数， 其实这是我们标准入口函数， 类似C语言的 main

topEntity
  :: Clock System
  -> Reset System
  -> Enable System
  -> Signal System (Signed 9, Signed 9)
  -> Signal System (Signed 9)
topEntity = exposeClockResetEnable mac

显然一个时序电路应该存在时钟系统，复位系统， 使能系统， 信号系统。

这里具有五个参数， exposeClockResetEnable ， 这个是一个暴露时钟，复位，使能系统的组件

对于 topEntity 标志 VHDL 的起点，topEntity 应该满足以下条件

  • 必须完全单态
  • 必须完全一阶
  • 尽管并非绝对必要，但建议公开隐藏的时钟和复位参数，因为这样可以更容易地在生成的 HDL 中进行用户控制的名称分配。

好吧， 现在你可以使用 :verilog 、 :vhdl 、:systemverilog 生成相应的文件

Verilator

这里是番外

生成的 verilog 文件如下

我们可以使用 Verilator 编译, 确保安装了 Verilator

首先编译 topEntity.v 文件， 输出 cpp 源码

verilator -cc topEntity.v

topEntity_shim.cpp 这个是模拟的程序文件, 参考 Verilator 官方文档进行修改， 比如输出时序图， 然后通过 GTKWare 查看， 这里是没有默认生成 vcd 文件的。

#include <cstdlib>

#include <verilated.h>

#include "VtopEntity.h"

int main(int argc, char **argv) {
  Verilated::commandArgs(argc, argv);

  VtopEntity *top = new VtopEntity;

  while (!Verilated::gotFinish()) {
    top->eval();
  }

  top->final();

  delete top;

  return EXIT_SUCCESS;
}

关于 Verilator 的简易教程可以参考 这里， 这小段也是参考这篇文章。

执行

verilator -Wall --trace -cc topEntity.v --exe topEntity_shim.cpp

然后使用 make 构建

make -C obj_dir -f VtopEntity.mk

他会默认构建 VtopEntity 这个可执行文件

./obj_dir/VtopEntity 运行程序， 目前来说程序一直执行， 并不会主动退出， 所以 C-c 退出吧， 本人学识有限， 目前还未会使用 Verilator
