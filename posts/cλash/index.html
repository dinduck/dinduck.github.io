当前配置

Archlinux

确保你安装了 make 和 base-devel

安装 stack

安装 stack

# Linux
curl -sSL https://get.haskellstack.org/ | sh

安装完成后， 我们不需要管太多事情

运行以下命令生成一个简单的 基础项目

stack new my-clash-project clash-lang/simple
cd my-clash-project

接着输入 stack build 构建这个基本项目

第一次运行需要很久， 需要下载编译器和相关库， 所以耐心等待即可。

默认会有一个示例文件 src/Example/Project.hs

module Example.Project (topEntity, plus) where

import Clash.Prelude

-- | Add two numbers. Example:
--
-- >>> plus 3 5
-- 8
plus :: Signed 8 -> Signed 8 -> Signed 8
plus a b = a + b

-- | 'topEntity' is Clash's equivalent of 'main' in other programming
-- languages. Clash will look for it when compiling 'Example.Project'
-- and translate it to HDL. While polymorphism can be used freely in
-- Clash projects, a 'topEntity' must be monomorphic and must use non-
-- recursive types. Or, to put it hand-wavily, a 'topEntity' must be
-- translatable to a static number of wires.
topEntity :: Signed 8 -> Signed 8 -> Signed 8
topEntity = plus

这是一个 8 位的加法器

运行 stack run clashi 进入交互解释器

使用 :load 加载 hs 文件， 这个交互解释器的使用不再赘述，学 haskell 时就知道了

我们在 src 新建 Mac.hs 文件

module MAC (topEntity, ma) where
import Clash.Prelude

ma acc (x, y) = acc + x * y

topEntity = ma

topEntity 类似 C语言的 main 函数， 显然我们的 MAC.hs

然后在 my-clash-project.cabal 文件 , 添加内容

运行 stack run clash -- MAC --verilog

会有特别多 Warning, 但是不影响编译通过。

然后在 verilog/MAC.topEntity/topEntity.v 看见我们的 verilog 代码

这里是64位的两输入, 输出128位的相加结果。因为这里默认的类型是 Integer

相应如果你想要 vhdl 代码可以使用 stack run clash -- MAC --vhdl
